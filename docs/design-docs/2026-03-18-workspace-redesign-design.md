# Workspace Redesign — v3 Rearchitecture

> CEO plan review output. Base: v2.15.17. Discards v3.0 Agent SDK work.
> Generated by /plan-ceo-review on 2026-03-18 | Mode: SELECTIVE EXPANSION

## Vision

Replace the fragmented 4-tab sidebar (Repos/Worktrees/Terminals/PRs) with a workspace-first model. The sidebar becomes a flat list of workspaces — no tabs at all. Each workspace is a directly-added folder that triggers git/gh integrations if it's a git repo. Clicking a workspace with no sessions shows a development dashboard (PRs, activity feed). Each session gets a dynamic PR top bar that changes its action button based on branch/PR/CI state, sending contextual prompts to the Claude terminal session. Multi-tab sessions per workspace for parallel conversations. Per-workspace settings.

Key constraint: NO Agent SDK — main session remains a terminal Claude Code instance. No structured chat UI. All richer functionality comes from workspace management, the dynamic PR top bar, and multi-tab session orchestration.

## Dynamic PR Top Bar State Machine

```
BRANCH STATE              PR STATUS     CI STATUS    ACTION BUTTON          COLOR
────────────────────────────────────────────────────────────────────────────────────
No commits ahead          —             —            (minimal/no bar)       gray
Commits ahead, no PR      —             —            [Create PR]            accent
PR exists                 Draft         —            [Ready for Review]     muted
PR exists                 Open          Passing      [Code Review]          green
PR exists                 Open          Failing      [Fix Errors] (N/M)    red/amber
PR exists                 Open          Pending      [Checks Running...]   amber
PR exists                 Merged        —            [Archive]              purple
PR exists                 Closed        —            [Archive]              gray
```

Action buttons send contextual prompts to the active Claude terminal session.
Archive = existing cleanup flow (kill session, remove worktree, prune branch).

## Full Feature Set

### Core Rearchitecture
- Replace 4-tab sidebar with flat workspace list (no sidebar tabs at all)
- Replace roots system with direct folder addition (workspace = folder)
- Workspace tree: always expanded, repos with sessions nested underneath
- Terminals are a session type under a workspace (different icon), not a separate section
- Context menu action on workspace to "Open terminal" at repo root
- Each workspace detects git repo → enables git/gh integrations
- Per-workspace settings (agent, yolo, continue, tmux, custom args, custom prompts)
- Multi-tab sessions: each tab = separate Claude Code terminal process

### Worktree Default Names — Mountains Theme
`+ new worktree` instantly creates a worktree with the next mountain name:
```
everest, kilimanjaro, denali, fuji, rainier, matterhorn, elbrus, aconcagua,
kangchenjunga, lhotse, makalu, cho-oyu, dhaulagiri, manaslu, annapurna,
nanga-parbat, olympus, mont-blanc, k2, vinson, erebus, logan, puncak-jaya,
wilhelm, cook, ararat, etna, shasta, whitney, hood
```
- Counter tracked per-workspace: `nextMountainIndex`
- Click `+ new worktree` → immediately creates worktree + starts session (no dialog)
- Branch auto-renamed after first message via prompt injection (see below)

### Branch Auto-Rename (Conductor pattern)
When the user sends their first message in a new worktree session:
1. Detect first Enter in `ws.ts` (buffer client→server data)
2. Prepend the branch rename prompt before the user's message:
   ```
   [Before responding: rename the current branch using `git branch -m` to a
   descriptive name based on this task. {user's branch rename preferences}.
   Then proceed with the task.]

   {user's actual message}
   ```
3. The Claude/Codex CLI session renames the branch itself
4. Session metadata updates when git refs change
5. Only happens once per worktree (flagged `needsBranchRename`)

### Per-Workspace Settings (Conductor-inspired)
Accessible via gear icon on workspace item. Full settings page per repo:

**Git settings:**
- Default branch to create worktrees from (auto-detected, configurable)
- Remote origin (default: `origin`)
- Branch prefix (e.g., `dy/`, `feat/`)

**Session defaults:**
- Default agent (claude / codex)
- Default continue, yolo, tmux toggles
- Custom claude args

**Custom prompts (collapsible text areas with preview):**
- **Code review preferences** — injected when "Code Review" action button clicked
- **Create PR preferences** — injected when "Create PR" action button clicked
- **Branch rename preferences** — injected with first message for auto-rename
- **General preferences** — injected at start of every new session

```typescript
interface WorkspaceSettings {
  // Session defaults
  defaultAgent?: AgentType;
  defaultContinue?: boolean;
  defaultYolo?: boolean;
  launchInTmux?: boolean;
  claudeArgs?: string[];

  // Git settings
  defaultBranch?: string;
  remote?: string;
  branchPrefix?: string;

  // Custom prompts
  promptCodeReview?: string;
  promptCreatePr?: string;
  promptBranchRename?: string;
  promptGeneral?: string;

  // Worktree naming
  nextMountainIndex?: number;
}
```

## Architecture

### Server Modules (off v2.15.17 base)
- `workspaces.ts` (NEW) — replaces roots: CRUD for workspace folders, git detection
- `git.ts` (ENHANCED) — activity feed, CI check status, branch info, branch switching
- `sessions.ts` (ENHANCED) — multi-session grouping by workspace path
- `config.ts` (ENHANCED) — per-workspace settings in config.json
- All other modules (pty-handler, ws, auth, etc.) — minimal changes

### Frontend Components (off v2.15.17 base)
**New:** WorkspaceItem, RepoDashboard, PrTopBar, SessionTabBar, SmartSearch, BranchSwitcher, empty state components
**Modified:** Sidebar (flat list), App (layout routing)
**Removed:** SessionFilters, PrRepoGroup, all v3.0 SDK components (ChatView, ChatInput, PermissionCard, cards/, sdk-handler.ts)

### API Changes
- `GET/POST/DELETE /workspaces` — replaces `/roots`
- `GET /workspaces/:path/dashboard` — aggregated dashboard data (PRs + activity)
- `GET/PATCH /workspaces/:path/settings` — per-workspace config
- `GET /workspaces/:path/ci-status` — CI check results for a branch
- `POST /workspaces/:path/branch` — switch branch (git checkout)
- Remove: `/roots`, `/repos` endpoints

## Visual Design System (v3 Terminal Aesthetic)

Inspired by Conductor's information density + tmux/terminal visual language.

  --status-warning: #fbbf24;  /* CI pending */

### Typography
- **Everything is monospace** — the entire app uses `--font-mono`
- Workspace names: `--font-size-lg`, `font-weight: 600`
- Session names, PR titles: `--font-size-base`
- Metadata (branch, time, diff stats): `--font-size-xs`, `var(--text-muted)`
- Tab labels: `--font-size-sm`

### Component Specs

**Sidebar workspace item:**
- Colored letter initial (first char of repo name) in monospace square
- Repo name: `--font-size-lg`, bold
- Indented sessions: status dot + name, `--font-size-sm`
- Quick-action icons on hover (desktop) / always visible (mobile)
- Solid `--border` separator between repos
- Bold text for unread/attention items

**PR top bar (36px height):**
- Background: `var(--surface)`
- Bottom border: `1px solid var(--border)`
- Branch text: `--font-size-sm`, `var(--text-muted)`, breadcrumb with `>`
- PR link: `var(--accent)`, monospace, `#` prefix
- Action button: pill shape, state-colored background, white text
  - Create PR → `var(--accent)` bg
  - Code Review → `var(--status-success)` bg, dark text
  - Fix Errors → `var(--status-error)` bg
  - Checks Running → `var(--status-warning)` bg, dark text
  - Archive (merged) → `var(--status-merged)` bg
  - Archive (closed) → `var(--border)` bg, muted text

**Session tab bar (32px height):**
- Active tab: `var(--surface)` bg, bottom border `var(--accent)`, `var(--text)`
- Inactive tab: transparent bg, `var(--text-muted)`
- Close ×: appears on hover (desktop), always visible (mobile), `--font-size-xs`
- + button: same style as inactive tab
- Overflow: horizontal scroll, no wrapping

**Dashboard:**
- PR rows: left-aligned title + metadata, right-aligned action button (pill)
- Activity rows: monospace commit hash `var(--text-muted)` + message `var(--text)` + branch name in parens
- Section headings: `--font-size-sm`, uppercase, `var(--text-muted)`, letter-spacing
- Solid `--border` separator between PR list and activity feed

**Smart search:**
- Input: monospace, 🔍 prefix or `>` prompt character (terminal style)
- Dropdown: matched workspace names, bold matching characters
- Background: `var(--surface)`, border: `var(--border)`

### Mobile Responsive

- **PR top bar**: Collapse to branch name + action button only. PR link/target behind context menu.
- **Session tabs**: Horizontal scroll, 44px touch targets, × always visible.
- **Sidebar tree**: Same layout in overlay drawer, larger touch targets (44px).
- **Dashboard**: Full-width PR cards, tappable rows, action buttons right-aligned.
- **Add workspace**: Text input with path autocomplete (no showDirectoryPicker on mobile).

### Accessibility

- Tab order: search → workspaces → sessions → main area → PR bar → tab bar → terminal
- `<nav aria-label="Workspaces">`, `role="tablist"` for tab bar
- Action buttons: descriptive `aria-label` per state (e.g., "Fix 2 of 8 failing checks")
- All status colors on #000 exceed WCAG AA 4.5:1 contrast
- Touch targets: 44px × 44px minimum
